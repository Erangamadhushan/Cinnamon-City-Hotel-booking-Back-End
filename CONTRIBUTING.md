# Contributing Guide

Thank you for your interest in contributing to this project! 🚀
We welcome contributions from the community.

## 1. Fork the Repository

Click the **Fork** button at the top right of this repository.

## 2. Clone Your Fork

```bash
git clone https://github.com/Erangamadhushan/Cinnamon-City-Hotel-booking-Back-End.git
```

## 3. Create a New Branch

Always create a new branch before making changes.

```bash
git checkout -b feature/your-feature-name
```

Examples:

```
feature/add-login-api
bug/fix-authentication-error
docs/update-readme
```

## 4. Make Your Changes

Make sure your code follows the project structure and coding style.

If possible:

* Write clean code
* Add comments if necessary
* Test your changes before submitting

## 5. Commit Your Changes

Use meaningful commit messages.

Example:

```
git commit -m "fix: resolve login API validation bug"
```

## 6. Push Your Branch

```bash
git push origin feature/your-feature-name
```

## 7. Create a Pull Request

Go to the original repository and open a **Pull Request**.

Please include:

* Clear description of the change
* Related issue number (if applicable)

Example:

```
Closes #12
```

## 8. Code Review

Your Pull Request will be reviewed.
Please be open to feedback and requested changes.

## Reporting Bugs

If you find a bug, please open an issue and include:

* Description of the bug
* Steps to reproduce
* Expected behavior
* Screenshots if possible

## Feature Requests

You can open an issue with the **feature request** label and explain the improvement.

---

Thank you for helping improve this project! 🎉
